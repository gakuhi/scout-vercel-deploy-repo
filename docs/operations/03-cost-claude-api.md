## 2. コスト上限（月次）

### 2.1 全体上限

プラン別の課金体系（#41）が未確定のため、現時点では **全体上限ベース** で運用。プラン体系確定後（#41 完了後）に再分配する。

| 項目 | しきい値（月次） | 用途 |
| :---- | :---- | :---- |
| **ハード上限** | **$300 / 月** | 暴走時に API キーを失効させる最終ライン |
| **警告ライン** | **$200 / 月** | 月内に上限超過する見込み |
| **早期警戒ライン** | **$100 / 月** | 想定の倍。原因確認 |
| **想定通常範囲** | **$50 以内 / 月** | MVP 後 1〜2 ヶ月の想定（学生 100〜300 名想定） |

### 2.2 数値の根拠

- **想定通常範囲 $50**: 学生 1 人あたりプロフィール統合 = 平均 0.15 USD × 300 名想定 = 45 USD。`force=true` の制限 (#234) を前提。
- **早期警戒 $100**: 想定の倍。利用増 or バグ・攻撃の早期検知。
- **警告 $200**: 月内推移で上限到達確実な水準。
- **ハード $300**: MVP 期の許容コスト最終ライン。これを超える前に運営判断（API キー失効・機能停止）。

> プロダクト成長に応じて、リリース 3 ヶ月後に再評価。

## 3. アラート発火条件

### 3.1 月次総コスト

| 段階 | Severity | 発火条件 | アクション |
| :---- | :---- | :---- | :---- |
| 早期警戒 | P2 | 当月累計コストが **$100 到達** | Slack 通知。原因確認 |
| 警告 | **P1** | 当月累計コストが **$200 到達** | Slack 通知 + 福田メンション。利用元の調査 |
| ハード | **P1** | 当月累計コストが **$300 到達** | Slack 通知 + メール。**API キー失効を運営判断** |
| 月次予測 | P2 | 月初 7 日時点の累計から **月内 $200 超過の予測** | Slack 通知 |

### 3.2 短時間スパイク（攻撃・バグ検知）

| 条件 | Severity | 想定原因 |
| :---- | :---- | :---- |
| 1 時間で **$10 超過** | **P1** | バグ・無限ループ・攻撃 |
| 1 学生の `force=true` リトライが **連続 5 回以上** | **P1** | #234 の攻撃面顕在化 |
| 1 日のリクエスト数が **想定の 5 倍超** | P2 | 想定外の利用パターン |

### 3.3 トークン異常検知

| 条件 | Severity | 想定原因 |
| :---- | :---- | :---- |
| 単一リクエストの input トークンが **50,000 超過** | P2 | プロンプトインジェクション or データ肥大化 |
| 単一リクエストの output が `max_tokens` に到達した割合 **> 5% / 24h** | P2 | `MAX_TOKENS` 不足。仕様見直し検討 |

## 6. 計測実装メモ（参考、実装は #45 / #46）

Claude API 呼び出しの構造化ログ例:

```ts
// 疑似コード。実装は #45 / #46
{
  level: "info",
  component: "claude-api",
  feature: "profile-sync",  // profile-sync / scout-draft / ...
  model: "claude-sonnet-4-6",
  input_tokens: 12345,
  output_tokens: 3456,
  cache_read_tokens: 0,
  cache_creation_tokens: 0,
  estimated_cost_usd: 0.123,
  duration_ms: 23456,
  status: "ok",
  request_id: "..."
}
```

`estimated_cost_usd` はアプリ層で input/output トークンと最新単価から算出する（単価表は環境変数 or 定数で管理）。

> プロンプトキャッシュ（cache_read / cache_creation）が有効化された場合は、これらも記録してコスト算出に反映する。
